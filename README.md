# 糖糖花园 · Sugar Garden

一个多家庭共用、云端同步的儿童糖分摄入记录工具。设计意图是**温柔**——不是控制孩子，而是帮家长和孩子一起"看见"每天吃了多少糖，让克制变成一种日常习惯。

**在线体验**：<https://qianyangpeng.github.io/sugar-garden/>

设计稿来自 [Claude Design](https://claude.ai/design)；后端 Cloudflare Workers + D1；前端单文件 React（Babel 浏览器内编译，零构建）。

## 功能详解

### 1. 身份 & 家庭

**首次打开** 显示欢迎页，两条路：

- 🌱 **开启新家庭**：四步引导
  1. 小朋友名字（最多 8 字）
  2. 生日 → 自动按年龄推荐每日糖分上限（WHO 标准：≤3 岁 15g、≤6 岁 19g、≤10 岁 24g、其他 30g）
  3. 每日上限调整（三档快选 + 手动输入 5-60g）
  4. 你是谁（爸爸/妈妈/外婆/奶奶 四快选 + 自定义），过一次 Turnstile 隐形验证
- 🔗 **加入家人的花园**：粘贴邀请链接（或打开别人发来的 `#join=...` 链接自动识别）+ 选自己的称呼

**家庭模型**：
- 每个家庭有唯一 `familyId` (UUID) + `token` (256-bit 随机)，两者都**在你手机上生成**
- 服务器只存 `sha256(token)`，永远看不到明文 token
- 家庭成员数量无上限（受写限流限制）

**邀请 = URL fragment**：
- 邀请链接形如 `https://.../sugar-garden/#join=<familyId>.<token>`
- `#` 后的部分浏览器**不会发送给服务器**，CF 访问日志、CDN、GitHub 访问记录都看不到 token
- 收到的人点链接 → app 自动识别 → 直跳"告诉我们你是谁"

**邀请链接泄露了怎么办**：
- 设置 → 危险操作 → **重新生成邀请链接**
- 服务器端的 token 哈希被替换，旧链接立刻失效，其他家人手上的 token 也会失效（需要你重新发邀请）

**离开家庭**：
- 设置 → 危险操作 → **退出家庭**
- 清本机 localStorage（家庭 ID/token/成员身份 + 本地状态）
- 云端数据保留（你老婆还能继续用）
- 可以重新开新家庭，或用新邀请链接加入别家

### 2. 每日记录

**今日屏幕**（默认首页）三大块：

**① 花苞卡片**
- 左边是今日正在长的花苞 SVG（轻微摇摆动画）
- 右边有"健康提示"（根据糖量/上限和时间比例动态变化）：
  - `thriving`（很少吃）：「XX 今天状态超好！花苞鼓鼓的～」
  - `ok`（刚好）：「XX 今天吃得刚好，花苞慢慢长」
  - `stressed`（吃多了）：「花苞有点担心…稍微省着吃哦」
  - `wilting`（快超标）：「糖快超标啦，花有点难过」
  - `dead`（严重超标 ≥1.3 倍）：「呜…今天糖糖太多啦，花蔫了」
- 大号数字：`已摄入 / 上限 g`
- 下面一行小字：「还能吃 Xg」或「超出 Xg」

**② 分段进度条**
- 水平进度条按上午（7:00-12:00）/下午（12:00-17:00）/晚上（17:00-22:00）等分为三段
- 紫色竖线标记「此刻应该的进度」（按当前时间线性映射）——早上吃多了就能看出来
- 每段下方显示该时段已吃克数
- 超标时进度条变红

**③ 记一笔**
- 大绿按钮「🍭 记一笔」弹出底部抽屉
- 旁边两个小按钮：📅 月历、📈 趋势、⚙ 设置

**食物选择器**（底部抽屉）：
- 分类 Tab：零食 / 饮料 / 水果 / 烘焙
- 24 种预设（每种带 emoji、名字、糖分克数）：
  - 零食：棒棒糖 6g、软糖 4g、巧克力 12g、饼干 5g、棉花糖 7g、冰淇淋 14g、布丁 10g、酸奶 8g
  - 饮料：果汁 18g、汽水 26g、奶茶小杯 22g、热可可 14g、运动饮料 12g
  - 水果（natural flag）：苹果半个 9g、香蕉 12g、葡萄 10g、草莓 4g、西瓜片 9g、橙子 12g
  - 烘焙：甜甜圈 15g、小蛋糕 20g、麦芬 17g、甜面包 8g、蜂蜜 17g
- 第 25 格「➕ 自定义」：输入名字 + 克数（用于"生日蛋糕""某款特殊饼干"之类）

**糖糖日记**（记录列表）：
- 今天已录入的每一笔都按时间倒序列出
- 每条显示：emoji 方块、食物名、时:分、克数（绿色）、删除按钮（×）
- 没录入时显示空状态「今天还没吃糖 ✨」

### 3. 在校糖分

专门为学龄儿童设计。用来处理"孩子在学校吃了点零食但你不知道具体吃了啥"的情况。

- **工作日自动亮起**（周一到周五）🎒：提示「今天有上学吗？」
- **周末**变成 🏠「在家度过 · 今天所有糖都记在这里」
- 点卡片弹出底部抽屉：六档快选（0/3/5/8/10/15g）或手动输入
- 在校糖分会从今天的额度里扣掉
- 点确定后，今日进度条、花苞状态、月历全部自动更新

### 4. 花园 & 花朵系统

**核心隐喻**：每天一朵花，糖越少花越好，坚持久了解锁稀有品种。

**花的品质**（由当日结束时 `糖量/上限` 比例决定）：
| 品质 | 比例 | 视觉 |
|------|------|------|
| `perfect` | <0.4 | 最大、鲜艳、饱满 |
| `great` | 0.4–0.7 | 标准开放 |
| `ok` | 0.7–0.9 | 茎微倾 |
| `okish` | 0.9–1.0 | 茎歪、花头转向 |
| `wilted` | 1.0–1.3 | 蔫花（去色、下垂） |
| `dead` | ≥1.3 | 只剩枯枝 + 旁边冒个绿芽（暗示"明天重来"） |

**三类花种（共 18 种）**：

**🌸 普通花（8 种）**——按当天品质和家庭种子随机抽选：
郁金香、雏菊、向日葵、罂粟、波斯菊、万寿菊、风铃草、三色堇

**✨ 稀有花（5 种）**——只有 `ratio < 0.5` 时有约 18% 概率触发（约每 5 个低糖日出一朵）：
- 月光花 — 银紫花瓣、带闪烁星星动画
- 水晶玫瑰 — 冰蓝色、棱角几何
- 彩虹百合 — 六色花瓣
- 星砂花 — 深蓝底带金色星粒
- 蜂蜜花 — 金色 + 旁边有只小蜜蜂

**🎖 解锁花（3 种）**——连续达标一定天数解锁，之后 50% 概率开：
- 翡翠花（连续 3 天达标）
- 樱花（连续 5 天达标）
- 凤凰花（连续 7 天达标）

**确定性花**：花种 RNG 的种子是 `familyId + date`，这意味着**所有家人、所有设备、所有时刻，同一天看到的都是同一朵花**。就算家人后来才同步到，他看到的依然一致。

**今天的花是"花苞"**：今天还没定型，显示一个绿色花苞（SVG），第二天早上打开才会"种下去"并揭晓品种。

### 5. 种花仪式

每天第一次打开 app（`lastOpenedDate != today`）时触发一段动画：

1. 绿渐变背景 + 小字「🌤️ 今天是新的一天」
2. 一只手（SVG）从画面顶部缓缓降下，把昨天的花植入土壤
3. 阳光光束从中心发散 + 金色星星闪烁
4. 揭晓昨天的花：「X 月 X 日开出了漂亮的花」＋ 花的名字（稀有会标 ✨，解锁会标 🎖）
5. 如果解锁了新品种：覆盖一层深绿蒙层，跳出"🎖 新品种解锁"弹窗，显示新花和解锁条件

**防轰炸**：如果多天没打开（比如旅行一周），队列里只展示**最近一天**的仪式，其他天悄悄种好。

**失败日**（花死/蔫）的文案是温柔的：
- dead：「X 月 X 日吃得有点多，不过明天又是新开始」
- wilted：「X 月 X 日的花蔫了一点点，没关系的」

### 10. 设置

顶部 ⚙ 进入。

**可编辑字段**：
- 小朋友名字
- 生日（显示当前年龄）
- 每日糖分上限（5-60g，WHO 参考值自动标出）

这些是**家庭级别**的设置，保存后会同步给所有家人。

**家庭成员面板**：
- 成员数量徽章
- 每人一张横条：首字母头像（绿底） + 称呼 + "最近活跃日期"
- 你自己的那条标注「· 你」
- 大紫色按钮「🔗 邀请家人」弹出邀请抽屉：
  - 显示完整邀请链接
  - "复制链接"按钮（成功复制后变绿「✓ 已复制」）
  - 安全提示：「拿到这个链接的人就能加入家庭。只发给信任的家人，别贴公开群。」

**我的花园统计**：
- 已种花总朵数
- 已解锁花种数（`N / 3`）

**危险操作区**（朱红虚线框）：
- **重新生成邀请链接**：服务器端 token 轮换，所有旧邀请链接立刻失效，**其他家人需要你重新发邀请**。操作前有确认弹窗。
- **退出家庭 / 换家庭**：清本机 localStorage，回到欢迎页。云端数据保留。

### 11. 同步 & 离线

**本地优先**：每次操作（记糖、删除、改设置、填在校糖量）先写 localStorage + 标 `_pending: true`，UI 立即反映。

**后台推送**：800ms 防抖后发到 Worker：
- `POST /entry`（新增或更新，按客户端 UUID 幂等）
- `DELETE /entry/:id`（软删除，标 `deleted_at`）
- `PUT /school-sugar`（按 `(family_id, date)` upsert）
- `PATCH /family`（家庭级设置）

**定时拉取**：每 30 秒 + 网络恢复时：
- `GET /family` 拉最新家庭设置 + 成员列表
- `GET /sync?since=<ts>` 拉所有 `updated_at > ts` 的 entries 和 school_sugar 增量
- 本地按 `updated_at` 做 last-write-wins 合并

**冲突解决**：
- entries 是 append-only + 客户端 UUID → 两人同时录入是两条独立行，零冲突
- 删除是软删除（`deleted_at`），同步时传播
- 其他字段（设置、在校糖量）按 `updated_at` 字段级 last-write-wins
- 同一天的 entries 变动会自动让"今日的花"失效重算（保证同步后花朵一致）

**离线体验**：
- 没网时 UI 完全正常（所有数据从 localStorage 读）
- `syncStatus` 变 `'offline'`
- 操作入队，网恢复自动 flush
- Service Worker 缓存了 shell，app 本身离线可开

**401 处理**：
- 如果服务器说 token 不存在（比如被家人 rotate 过），自动清理本机身份和状态，回到欢迎页

### 12. PWA 安装

`manifest.json` + `sw.js` 让它能"添加到主屏幕"：

- **iOS Safari**：分享 → 添加到主屏幕 → 看到「糖糖花园」图标，全屏运行不含 Safari UI
- **Android Chrome**：地址栏右侧"安装"提示或菜单里手动装 → 出现在抽屉里
- 装完后：离线也能开、没有浏览器地址栏、像原生 app

Service Worker 策略：
- 壳资源（index.html, src/*.jsx, icon.svg, manifest）缓存优先 + 网络更新
- API 请求（跨域到 Worker）永远走网络
- 壳变更时 bump `VERSION` 字符串自动清旧缓存

## 隐私 & 安全

### 数据流里看得见的和看不见的

| 谁能看到 | 什么 |
|---|---|
| 你的手机 | 完整数据（localStorage 明文） |
| 你家人的手机 | 完整数据（localStorage 明文，同一家庭） |
| CF Worker 日志 | API 调用 URL、IP、时间戳。**看不到** token 明文、**看不到**邀请链接（都在 URL fragment 里） |
| CF D1 数据库 | `sha256(token)`、家庭设置、每条 entry 明文、成员姓名。**看不到** token 明文 |
| 部署者（你）在 CF dashboard | D1 里的所有数据（可读） |
| 第三方（攻击者） | 啥都看不到，除非：拿到某台手机 unlocked、拦截到 HTTPS、或者你把邀请链接贴到公开群 |

### 关键设计原则

- **Token 客户端生成**：用 `crypto.getRandomValues(32 bytes)` 生成，永远不离开浏览器（除非你自己复制/分享）
- **Token 服务器只存哈希**：SHA-256，单向，从 D1 反推不出明文
- **邀请走 URL fragment**：浏览器标准规定 `#` 后内容不发到服务器
- **鉴权常量时间比较**：防时序攻击（低威胁场景，但零成本）
- **CORS 限来源**：只允许 `ALLOWED_ORIGIN`（你配的 Pages 域名）
- **HTTPS 强制**：Workers 自带，Pages 也默认

### 刻意没做的

- **端到端加密**：数据在 D1 里是明文。部署者（你）如果愿意花时间查 D1 能看到每家吃了啥。原因：四岁吃糖不是医疗数据，E2E 复杂度收益比低。如果你想做，是后续工作——AES-GCM、密钥从 token 派生、邀请 URL 里附密钥。
- **多因素/密码登录**：用 token 模型简化心智，不搞账号系统。
- **审计日志**：D1 不存"谁什么时候点了什么"，只存状态。

### 防滥用（后端）

- **Turnstile**（注册）：CF 自家 CAPTCHA，挡大部分脚本化注册
- **每 IP 限流**：5 次注册/分钟（防单机刷）
- **每家限流**：300 次写/小时（防 token 泄露后被刷）
- **全局熔断**：单日注册数超 `MAX_REGISTRATIONS_PER_DAY`（默认 500）直接 503
- **Cron 清理**：60 天无活动的家庭每周一自动删除

## 技术架构

### 总图

```
手机 (iOS/Android)
  ├─ Safari/Chrome 打开 https://qianyangpeng.github.io/sugar-garden/
  ├─ 前端：单页 React（Babel 浏览器内编译，无构建步骤）
  ├─ 本地状态：localStorage（sg:state, sg:identity, sg:pending）
  └─ 邀请链接走 URL fragment (#join=…) —— 服务器日志看不到
         ↕ HTTPS
Cloudflare Worker (edge, 全球 300+ 节点)
  ├─ 路由：/family, /member, /entry, /school-sugar, /sync, /health
  ├─ 鉴权：Bearer <familyToken>  →  服务器只存 sha256
  ├─ 防滥用：Turnstile (注册) + Rate Limit (IP 5/min, family 300/hr)
  └─ Cron 每周一清理 60 天无活动的家庭
         ↕
Cloudflare D1 (SQLite at edge)
  families · members · entries · school_sugar
```

### 为什么选这些

- **Cloudflare Workers**：免费 10 万请求/天对家用足够；全球边缘节点延迟低；无服务器运维
- **D1 (SQLite)**：强一致性 + SQL 增量同步 (`WHERE updated_at > ?`) 比 KV 干净；免费 500 万行读/天
- **React + Babel CDN**：设计稿保真；零构建零工具链；家用规模无需优化
- **Append-only entries**：客户端 UUID → 两人同时录入零冲突；删除是软删除通过同步传播
- **确定性花朵**：种子 `familyId + date`，同一家庭所有设备同一天看到同一朵

## 部署自己的一份

**详细步骤看 [HANDOFF.md](./HANDOFF.md)** —— 7 步复制粘贴命令，20-30 分钟，全程免费档。

简要：
1. 注册 Cloudflare 账号，装 `wrangler`
2. 创建 D1 数据库 + KV 命名空间，ID 填进 `worker/wrangler.toml`
3. 在 Cloudflare Turnstile 创建站点，`wrangler secret put TURNSTILE_SECRET`
4. `wrangler deploy` 得到 Worker URL
5. 把 Worker URL + Turnstile siteKey 填进 `index.html` 里的 `SG_CONFIG`
6. push 到 GitHub，开启 Pages
7. 真机测一下
